# Doctrine of Pathfinder Aperture

## Sovereign Decision Standards

The Pathfinder Aperture is not an automated agent; it is an instrument of operator illumination. The following seven immutable doctrines govern all systems, modules, and extensions developed within this workspace:

---

### Doctrine 1: The Operator is the Source
All cognitive pipelines, evaluation matrix results, and recommended postures are auxiliary guides. No executive action can be initiated, modified, or completed without direct human authorization. There are no "autonomous" or "delegated" dispatch modes.

### Doctrine 2: Evidence is Non-Fungible
A claim without a direct, verifiable quote and a specific document anchor is classified as an **assumption**. Assumptions must be visually and functionally distinct from **evidence**. No system is permitted to treat inferred likelihoods or model probabilities as factual anchors.

### Doctrine 3: High-Velocity Grounding (RAPIDS-to-Coinbase Verification)
All digital asset coordinates and baseline metrics must correspond to live index points:
*   Real-time pricing feeds must leverage direct API sources (e.g., Coinbase Spot API).
*   Indirect variables (indices, commodities, speculative data) must be grounded via live query tools.
*   Fallback metrics must be clearly flagged to the operator with appropriate provenance tags.

### Doctrine 4: Friction must be Preserved
A friction point surfaced by the JEMMA Verator must never be auto-mitigated or swept under a software design element. If there is a dispute between historical database states and real-time feeds, the friction loop must remain actively locked in the JEMMA dispute ledger until the operator manually override or reconcile the telemetry discrepancy.

### Doctrine 5: Representational Drift Defense
Interfaces must not "administrative-ize" deep cognitive processes. Personnel dashboards, clean list controls, and standard forms are prohibited from masking the raw, high-velocity geometry of underlying cognitive spectrum arrays. Let the data shape remain topological, spatial, and clear.

### Doctrine 6: Substrate Delta Persistence
All verified learning, cognitive rules, and localized insights of an operation are written back into the permanent operator ledger (Firestore delta registry). When sessions reset, the operator’s learned principles must persist to construct a permanent substrate profile of acquired understanding.

### Doctrine 7: The Cryptographic Gate
No action or command payload can pass the Crystal Bridge without a cryptographic signature verification stamp and an operator consent code. The system does not write to external containers directly; it projects a secure, verified dossier ready for operator-driven implementation.
